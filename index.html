核心特性

基于 README 的产品设定：多项目管理、容器/环境编排、模板与脚手架、命令行驱动、可扩展插件。以下为首屏即可理解的价值卡片。

一键就绪的开发环境

用声明式配置拉起容器、依赖与常用工具，人人即开即用。

多项目看板

在一个面板里集中启动/停止/重建多个服务或项目。

模板与脚手架

内置常用栈模板，复制即可用，减少“环境折腾”。

CLI‑first

统一 codie 命令，覆盖初始化、运行、诊断与清理。

插件与扩展

以插件扩展能力：集成 CI/CD、代码质量、包管理、镜像仓库等。

团队协作与可复制性

让新人 10 分钟跑通；让跨平台协作不再受环境影响。

如何运作

1
声明项目

在 codie.config.* 中声明服务、容器与依赖；选择模板或自定义镜像。

2
一键启停

用 UI 或 codie up/down 管理多项目；支持 rebuild、logs、health。

3
共享复用

将配置与模板入库：同栈项目复用，同步到团队知识库。

CLI

# install (example) npm i -g @codie/cli # initialize a workspace codie init # bring services up codie up # or with a profile codie up --profile web+db # rebuild with cache busting codie rebuild --service api --no-cache # stop & clean codie down && codie clean

FAQ

这个名字会不会“过于可爱”？

我们希望它是亲切又专业的：以代码为“父”的敬畏感 + 让人微笑的趣味性。

和 devcontainer/docker‑compose 等怎么配合？

Codie 是更上层的编排与模板工具：读写这些标准文件，提供更友好的多项目体验与脚手架。

适合个人还是团队？

都适合。个人更快上手，团队获得“可复制的环境”和稳定交付。

支持哪些语言与框架？

开箱即用：Node/Python/Java/Go/Rust/前端工程等；也可通过模板与自定义镜像扩展。
